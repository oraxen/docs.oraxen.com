# Advanced Pack Generation

Advanced configuration options for resource pack generation, validation, optimization, and atlas management.

## Overview

Beyond basic pack generation settings, Oraxen provides advanced options for:
- Pack file validation and error handling
- Unicode/Force Unicode compatibility
- Texture format conversion (1.20.2+ atlas system)
- Atlas generation and optimization
- Custom file exclusions

**Location:** `settings.yml` → `Pack.generation`

---

## Pack Validation

### verify_pack_files

Validates all textures and models to ensure they follow valid resource pack formatting.

```yaml
Pack:
  generation:
    verify_pack_files: true  # Enable validation (default: true)
```

**What it checks:**
- Texture file formats (PNG validity)
- Model JSON syntax
- File naming conventions (lowercase, no spaces)
- Required model properties
- Texture dimensions (power of 2 for some formats)

**When enabled:**
- ✅ Catches errors before pack is generated
- ✅ Prevents malformed files from breaking the pack
- ✅ Logs detailed warnings for invalid files
- ❌ Slightly slower pack generation

**When disabled:**
- ✅ Faster pack generation
- ❌ Invalid files may break the resource pack
- ❌ Harder to debug issues

**Recommendation**: Keep `true` unless you're certain all your assets are valid.

### exclude_malformed_from_atlas

Prevents malformed textures from being included in the texture atlas.

```yaml
Pack:
  generation:
    atlas:
      exclude_malformed_from_atlas: true  # Default: true
```

**What it does:**
- Checks texture files for corruption
- Excludes broken textures from atlas.png
- Prevents pack from failing to load due to one bad texture

**When enabled:**
- ✅ Pack still works even with some broken textures
- ✅ Logs which textures were excluded
- ✅ Allows fixing textures without breaking entire pack

**When disabled:**
- ❌ One broken texture can corrupt the entire atlas
- ❌ Pack may fail to load

**Recommendation**: Always keep `true` for production servers.

## Unicode & Font Compatibility

### fix_force_unicode_glyphs

Fixes glyphs not showing when "Force Unicode" is enabled in Minecraft client settings.

```yaml
Pack:
  generation:
    fix_force_unicode_glyphs: true  # Default: true
```

**The Problem:**
- Minecraft's "Force Unicode" option (in Language settings) forces all text to use the Unicode font
- Without this fix, custom glyphs don't render when Force Unicode is ON
- Affects emojis, custom fonts, and interface glyphs

**How it works:**
- Generates additional font variants compatible with Force Unicode
- Adds fallback font definitions
- Ensures glyphs render in both normal and Unicode forced modes

**When enabled:**
- ✅ Glyphs work with Force Unicode ON and OFF
- ✅ Better player experience (some languages require Force Unicode)
- ❌ Slightly larger pack size

**When disabled:**
- ✅ Smaller pack size
- ❌ Glyphs don't work when Force Unicode is enabled
- ❌ Players using Asian languages may have issues

**Recommendation**: Keep `true` for international servers.

## Texture Format Conversion

### texture_slicer

Converts textures to the new 1.20.2+ atlas format by slicing oversized textures.

```yaml
Pack:
  generation:
    texture_slicer: true  # Default: true
```

**What it does (Minecraft 1.20.2+):**
- Automatically slices textures larger than 16x16
- Converts old "tall" texture formats to new sprite-based system
- Ensures compatibility with modern Minecraft versions

**Example:**
- **Old format**: 16x64 texture for animation (4 frames stacked vertically)
- **New format**: 4 separate 16x16 sprites + mcmeta animation definition

**When enabled:**
- ✅ Textures work on 1.20.2+
- ✅ Backward compatible with older versions
- ✅ Follows modern resource pack standards

**When disabled:**
- ❌ Large textures may not display correctly on 1.20.2+
- ❌ Animation textures may break

**Recommendation**: Keep `true` for 1.20.2+ servers.

## Atlas Generation

The atlas system controls how Minecraft combines multiple textures into a single texture atlas for performance.

### generate

Enable or disable atlas generation.

```yaml
Pack:
  generation:
    atlas:
      generate: true  # Default: true
```

**What is an atlas?**
- Combines many small textures into one large texture sheet
- Reduces GPU texture switches = better performance
- Minecraft automatically uses atlases for blocks, items, etc.

**When enabled:**
- ✅ Better performance (fewer texture switches)
- ✅ Standard Minecraft behavior

**When disabled:**
- ❌ Worse performance
- ❌ May cause rendering issues

**Recommendation**: Always keep `true`.

### type

Atlas generation strategy.

```yaml
Pack:
  generation:
    atlas:
      type: "SPRITE"  # Options: SPRITE or DIRECTORY
```

**SPRITE** (default, recommended):
- Generates atlas from individual sprite definitions
- More control over what's included
- Modern approach (1.19+)

**DIRECTORY**:
- Generates atlas from all textures in a directory
- Simpler but less control
- Legacy approach

**Recommendation**: Use `SPRITE` for modern packs.

## Model Path Behavior

### auto_generated_models_follow_texture_path

Controls whether auto-generated models are placed following texture directory structure.

```yaml
Pack:
  generation:
    auto_generated_models_follow_texture_path: false  # Default: false
```

**When false (default):**
```
textures/items/weapons/legendary_sword.png
models/items/legendary_sword.json  # Model in flat items/ directory
```

**When true:**
```
textures/items/weapons/legendary_sword.png
models/items/weapons/legendary_sword.json  # Model follows texture path
```

**Use cases:**

**Keep `false` when:**
- You want all models in a flat structure
- Simpler organization for small packs
- Default Oraxen behavior

**Set `true` when:**
- You have many items organized in subdirectories
- You want models next to their textures conceptually
- Easier to find related files

## File Exclusions

### excluded_file_extensions

Exclude specific file types from the final resource pack ZIP.

```yaml
Pack:
  generation:
    excluded_file_extensions:
      - ".psd"
      - ".xcf"
      - ".kra"
      - ".blend"
      - ".tmp"
```

**Common exclusions:**

| Extension | Description | Why exclude? |
|-----------|-------------|--------------|
| `.psd` | Photoshop files | Source files, not needed in pack |
| `.xcf` | GIMP files | Source files, large |
| `.kra` | Krita files | Source files, large |
| `.blend` | Blender files | 3D source files, very large |
| `.tmp` | Temp files | Not needed, may cause issues |
| `.bak` | Backup files | Duplicates, increase pack size |
| `.md` | Markdown docs | Documentation, not needed in pack |

**Example configuration:**
```yaml
Pack:
  generation:
    excluded_file_extensions:
      # Image editing source files
      - ".psd"
      - ".xcf"
      - ".kra"
      - ".ai"

      # 3D modeling files
      - ".blend"
      - ".obj"
      - ".fbx"

      # Development files
      - ".tmp"
      - ".bak"
      - ".old"
      - ".backup"

      # Documentation
      - ".md"
      - ".txt"
      - ".doc"
```

**Benefits:**
- ✅ Smaller pack size
- ✅ Faster download for players
- ✅ Cleaner pack structure
- ✅ No source files leaked to players

Optimal settings for a production server:

```yaml
Pack:
  generation:
    generate: true

    verify_pack_files: true
```

Standard settings work well:

```yaml
Pack:
  generation:
    verify_pack_files: true
    fix_force_unicode_glyphs: true
    texture_slicer: true
    atlas:
      generate: true
      type: "SPRITE"
```

```yaml

  generation:
    verify_pack_files: true
    fix_force_unicode_glyphs: true
    texture_slicer: true
    compression: BEST_COMPRESSION
```

## Troubleshooting

### "Pack generation is slow"

**Solutions:**
- Disable `verify_pack_files` if you're sure your assets are valid
- Reduce number of items
- Exclude more file extensions
- Check for very large textures (>512x512)

### "Some textures don't show up"

**Check:**
- `verify_pack_files: true` - Check console for errors
- `exclude_malformed_from_atlas: true` - Check if textures were excluded
- Texture file format (must be PNG)
- Texture file names (must be lowercase, no spaces)

### "Glyphs don't show with Force Unicode enabled"

**Solution:**
```yaml
fix_force_unicode_glyphs: true
```

Then regenerate the pack: `/oraxen reload all`

### "Pack is too large (>100MB)"

**Solutions:**
- Add more extensions to `excluded_file_extensions`
- Use `compression: BEST_COMPRESSION`
- Optimize textures:
   - Reduce resolution where possible
   - Use 8-bit PNG instead of 24-bit
   - Remove unused textures
- Enable `protection: false` temporarily to check uncompressed size

### "Textures look wrong on 1.20.2+"

**Solution:**
```yaml
texture_slicer: true
```

Ensure you're using the modern texture format.

### "Models not generating in expected locations"

**Check:**
```yaml
auto_generated_models_follow_texture_path: false  # or true based on preference
```

Verify your texture paths match your expected model paths.

For testing and iteration:

```yaml
Pack:
  generation:
    verify_pack_files: true  # Catch errors early
    fix_force_unicode_glyphs: true

    protection: false  # Can extract pack to debug
    excluded_file_extensions:
```

```yaml
Pack:
  generation:
```
