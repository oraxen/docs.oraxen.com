# mechanics.yml Configuration

Complete guide to configuring Oraxen's 44 built-in mechanics system. Enable, disable, and configure mechanics to add custom gameplay features.

## Overview

The `mechanics.yml` file controls all plugin mechanics - gameplay features that can be applied to items, blocks, and furniture.

**Location:** `plugins/Oraxen/mechanics.yml`

**Format:**
```yaml
mechanic_name:
  enabled: true/false  # Enable or disable this mechanic
  # Additional settings specific to each mechanic
```

## Mechanics Categories

Oraxen organizes 44 mechanics into 5 categories:

| Category | Count | Purpose |
|----------|-------|---------|
| **Miscellaneous** | 9 | Core item behaviors (commands, consumable, soulbound, etc.) |
| **Gameplay** | 13 | Blocks, furniture, durability, repair systems |
| **Combat** | 7 | Weapon abilities (thor, fireball, lifeleech, etc.) |
| **Farming** | 6 | Resource gathering (harvesting, smelting, bigmining, etc.) |
| **Cosmetic** | 5 | Visual effects (hats, skins, auras, backpacks) |

## Miscellaneous Mechanics

### commands
Execute custom commands when items are used.

```yaml
commands:
  enabled: true
```

**Usage**: Apply to items via `Mechanics.custom` with event triggers.

**Example**:
```yaml
magic_wand:
  Mechanics:
    custom:
      one_usage:
        event: CLICK:right:all
        actions:
          console:
            - "give %player% diamond 1"
```

### armor_effects
Apply potion effects when wearing armor pieces.

```yaml
armor_effects:
  enabled: true
  delay_in_ticks: 20  # Check interval for armor effects
```

**Settings**:
- `delay_in_ticks` - How often to check for armor and apply effects (default: 20 = 1 second)

**Usage**: Add potion effects to armor items.

**Example**:
```yaml
speed_boots:
  Mechanics:
    armor_effects:
      - SPEED:1
```

### consumable
Make items consumable (eat/drink) with custom effects.

```yaml
consumable:
  enabled: true
```

**Usage**: Make items edible with custom results.

### consumable_potion_effects
Apply potion effects when items are consumed.

```yaml
consumable_potion_effects:
  enabled: true
```

**Usage**: Add potion effects to consumable items.

**Example**:
```yaml
magic_apple:
  Mechanics:
    consumable:
      effects:
        - REGENERATION:5:60
        - ABSORPTION:2:120
```

### custom
Advanced custom action system with events, conditions, and commands.

```yaml
custom:
  enabled: true
```

**Usage**: Create complex item behaviors with:
- Event triggers (CLICK, BREAK, PLACE, KILL, etc.)
- Conditions (permissions, locations)
- Actions (commands, sounds, messages)
- Cooldowns

**Example**:
```yaml
teleport_stick:
  Mechanics:
    custom:
      teleport:
        event: CLICK:right:all
        cooldown: 10
        actions:
          player:
            - "spawn"
```

### itemtype
Set custom item types for compatibility with other plugins.

```yaml
itemtype:
  enabled: true
```

**Usage**: Define item categories (SWORD, HELMET, etc.) for plugin integrations.

### soulbound
Prevent items from being dropped on death.

```yaml
soulbound:
  enabled: true
```

**Usage**: Make items stay in inventory when player dies.

**Example**:
```yaml
legendary_sword:
  Mechanics:
    soulbound:
      enabled: true
```

### backpack
Portable storage system - functional backpacks.

```yaml
backpack:
  enabled: true
```

**Usage**: Create items that open custom GUIs for storage.

**Note**: Different from `backpack_cosmetic` (cosmetic-only).

### music_disc
Create custom music discs for jukeboxes.

```yaml
music_disc:
  enabled: true
```

**Usage**: Combine with `sound.yml` jukebox songs.

**Example**:
```yaml
epic_disc:
  material: MUSIC_DISC_13
  Components:
    jukebox_playable:
      song_key: "oraxen:epic_theme"
```

### misc
Miscellaneous features not fitting other categories.

```yaml
misc:
  enabled: true
```

## Gameplay Mechanics

### custom_block_sounds
Controls sound system for custom blocks and furniture.

```yaml
custom_block_sounds:
  noteblock_and_block: true       # Enable sounds for NoteBlock mechanic
  stringblock_and_furniture: true # Enable sounds for StringBlock & furniture
  chorusblock: true                # Enable sounds for ChorusBlock mechanic
```

**Important**: If both mechanics are disabled, sounds are forced off for those types.

**When to disable**:
- Performance optimization on servers with many custom blocks
- Conflicts with other sound plugins
- Debugging sound issues

### noteblock
Use NoteBlocks for custom blocks (fastest, most compatible).

```yaml
noteblock:
  enabled: true
  tool_types:  # Tools that can break custom blocks
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
  farmblock_check_delay: 1000  # Ticks between dryout checks for farmblocks
  remove_mineable_tag: false   # Prevent axes from fast-breaking custom blocks
```

**Key Settings**:

**`remove_mineable_tag`** (default: `false`)
- When `false`: Axes break NoteBlocks quickly (vanilla behavior)
- When `true`: Removes the `mineable/axe` tag, preventing fast breaking
- **Use case**: Set to `true` if players are breaking custom blocks too quickly with axes

**`farmblock_check_delay`** (default: `1000`)
- Interval in ticks for checking if farmblocks should dry out
- Higher = less frequent checks = better performance
- Lower = more responsive drying = worse performance

### stringblock
Use tripwire strings for custom blocks (supports saplings, growth).

```yaml
stringblock:
  enabled: true
  tool_types:
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
  sapling_growth_check_delay: 4000  # Ticks between sapling growth checks
  disable_vanilla_strings: false     # Disable vanilla tripwire placement
```

**Key Settings**:

**`disable_vanilla_strings`** (default: `false`)
- When `true`: Players cannot place vanilla tripwire strings
- When `false`: Vanilla strings work normally
- **Use case**: Set to `true` to reserve tripwire for custom blocks only

**`sapling_growth_check_delay`** (default: `4000`)
- How often to check if custom saplings should grow
- Affects custom tree growth mechanics

### chorusblock
Use chorus plants for custom blocks (natural appearance).

```yaml
chorusblock:
  enabled: true
  tool_types:
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
```

**Usage**: Best for decorative/natural-looking blocks.

### shaped_block
Use waxed copper for custom stairs, slabs, doors, trapdoors.

```yaml
shaped_block:
  enabled: true
  tool_types:
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
  convert_vanilla_waxed: true       # Convert vanilla waxed copper to unwaxed
  handle_world_generation: true     # Handle waxed copper in generated structures
```

**Key Settings**:

**`convert_vanilla_waxed`** (default: `true`)
- Converts existing waxed copper blocks to unwaxed versions
- Reserves waxed copper materials exclusively for custom blocks
- Prevents oxidation on converted blocks via marker system

**`handle_world_generation`** (default: `true`)
- Processes waxed copper in naturally generated structures (Trial Chambers, etc.)
- Ensures custom blocks work in generated structures

### block
Legacy mushroom stem blocks (deprecated - use noteblock instead).

```yaml
block:
  enabled: false  # Disabled by default
  tool_types:
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
```

**Note**: This is the old block system. Use `noteblock` for new projects.

### furniture
Custom 3D furniture using Display Entities or Item Frames.

```yaml
furniture:
  enabled: true
  default_furniture_type: DISPLAY_ENTITY  # DISPLAY_ENTITY, ITEM_FRAME, GLOW_ITEM_FRAME
  tool_types:
    - WOODEN
    - STONE
    - IRON
    - GOLDEN
    - DIAMOND
    - NETHERITE
  evolution_check_delay: 200    # Ticks between plant growth checks
  detect_viabackwards: true     # Block Display Entities on clients <1.19.4
```

**Key Settings**:

**`default_furniture_type`** (default: `DISPLAY_ENTITY`)
- `DISPLAY_ENTITY` - Modern system (1.19.4+), best visuals
- `ITEM_FRAME` - Legacy system, better compatibility
- `GLOW_ITEM_FRAME` - Glowing variant of ITEM_FRAME

**`detect_viabackwards`** (default: `true`)
- When `true`: Blocks Display Entity furniture for clients <1.19.4 via ViaBackwards
- Prevents crashes/issues on older clients
- **Use case**: Keep `true` if you have ViaBackwards and support <1.19.4 clients

**`evolution_check_delay`** (default: `200`)
- How often to check if furniture with plant mechanics should grow
- Used for custom plant furniture

### durability
Custom durability system for items.

```yaml
durability:
  enabled: true
```

**Usage**: Set custom durability values on items.

**Example**:
```yaml
crystal_sword:
  Mechanics:
    durability:
      value: 500
```

### efficiency
Mining speed modifiers for tools.

```yaml
efficiency:
  enabled: true
```

**Usage**: Make tools mine faster or slower.

**Example**:
```yaml
super_pickaxe:
  Mechanics:
    efficiency:
      amount: 10
```

### repair
Custom item repair system.

```yaml
repair:
  enabled: true
  oraxen_durability_only: false  # Only repair Oraxen items with custom durability
```

**Key Settings**:

**`oraxen_durability_only`** (default: `false`)
- When `true`: `/oraxen repair` command only repairs Oraxen items with custom durability
- When `false`: Command repairs vanilla items too
- **Use case**: Set to `true` to prevent repairing vanilla items

### food
Custom food items with saturation and nutrition.

```yaml
food:
  enabled: true
```

**Usage**: Create custom food items.

### toggle_light
Items that emit light when held/placed.

```yaml
toggle_light:
  enabled: true
```

**Usage**: Create portable light sources.

## Cosmetic Mechanics

### aura
Particle effects around players.

```yaml
aura:
  enabled: true
```

**Usage**: Add particle auras to items or as player effects.

### backpack_cosmetic
Visual backpacks on player's back (cosmetic only, no storage).

```yaml
backpack_cosmetic:
  enabled: true
```

**Note**: Uses packet-based armor stands for display. Different from `backpack` (which provides storage).

### hat
Wear items as hats.

```yaml
hat:
  enabled: true
```

**Usage**: Allow players to equip items as cosmetic hats.

### skin
Custom item skins/textures.

```yaml
skin:
  enabled: true
```

**Usage**: Apply different textures to items dynamically.

### skinnable
Items that can have their appearance changed.

```yaml
skinnable:
  enabled: true
```

**Note**: Different from `skin` - `skinnable` marks items as customizable, `skin` applies skins.

## Farming Mechanics

### bigmining
Mine multiple blocks at once (vein miner).

```yaml
bigmining:
  enabled: true
  call_events: true  # Whether Oraxen calls BlockBreakEvent or just listens
```

**Key Settings**:

**`call_events`** (default: `true`)
- When `true`: Oraxen calls `BlockBreakEvent` for each block broken
- When `false`: Oraxen only listens to events
- **Known issue**: Some enchantment plugins have issues when `true`
- **Use case**: Set to `false` if you have enchantment plugin conflicts

### harvesting
Auto-replant crops when harvesting.

```yaml
harvesting:
  enabled: true
```

**Usage**: Tools that automatically replant farmland.

### smelting
Auto-smelt mined blocks.

```yaml
smelting:
  enabled: true
  blacklist_cooked:  # Items that should NOT be auto-smelted
    - WET_SPONGE
```

**Key Settings**:

**`blacklist_cooked`** - List of items to never auto-smelt
- Default: `WET_SPONGE` (smelting would turn it into regular sponge)
- Add items here if smelting causes issues

**Example**:
```yaml
smelting:
  blacklist_cooked:
    - WET_SPONGE
    - ICE  # Don't melt ice
```

### watering
Hydrate farmland in an area.

```yaml
watering:
  enabled: true
```

**Usage**: Tools that water crops.

### bottledexp
Store and use XP in bottles.

```yaml
bottledexp:
  enabled: true
  durability_cost: 10  # Durability consumed per use
```

**Key Settings**:

**`durability_cost`** (default: `10`)
- How much durability is consumed when using bottled XP
- Set to `0` for no durability cost

### bedrockbreak
Break bedrock with custom tools.

```yaml
bedrockbreak:
  enabled: true
  disable_on_first_layer: false  # Prevent breaking bottom layer of bedrock
  durability_cost: 500           # Durability consumed per bedrock block
```

**Key Settings**:

**`disable_on_first_layer`** (default: `false`)
- When `true`: Cannot break Y=0 bedrock (prevents breaking into void)
- When `false`: Can break all bedrock layers
- **Important**: Set to `true` to prevent players from falling into void

**`durability_cost`** (default: `500`)
- Durability consumed when breaking one bedrock block
- High cost prevents excessive bedrock mining

## Mechanic Combinations

Some mechanics can be combined on the same item:

**Common Combinations**:
```yaml
legendary_pickaxe:
  Mechanics:
    durability:
      value: 5000
    efficiency:
      amount: 15
    bigmining:
      vein_miner: true
      blocks: 10
    smelting: {}
```

**Incompatible Combinations**:
- Cannot combine multiple block mechanics (noteblock, stringblock, chorusblock) on same item
- Cannot combine `backpack` and `backpack_cosmetic` (use one or the other)
- Cannot combine `skin` and `skinnable` in conflicting ways

## Performance Optimization

### High Performance Settings

For servers with limited resources:

```yaml
# Disable cosmetic mechanics
aura:
  enabled: false
backpack_cosmetic:
  enabled: false
hat:
  enabled: false

# Increase check delays
armor_effects:
  delay_in_ticks: 40  # Check less frequently (2 seconds)

noteblock:
  farmblock_check_delay: 2000  # Check less often

stringblock:
  sapling_growth_check_delay: 8000

furniture:
  evolution_check_delay: 400
```

### Low Resource Usage

Minimal mechanics for lightweight servers:

```yaml
# Only enable essential mechanics
durability:
  enabled: true
repair:
  enabled: true
custom:
  enabled: true

# Disable everything else
aura:
  enabled: false
# ... etc
```

## Troubleshooting

### "Mechanic not working"
1. Verify `enabled: true` in mechanics.yml
2. Check item has the mechanic properly configured
3. Reload: `/oraxen reload all`
4. Check console for errors

### "Custom blocks breaking too fast with axes"
```yaml
noteblock:
  remove_mineable_tag: true
```

### "Enchantment plugin conflicts with bigmining"
```yaml
bigmining:
  call_events: false
```

### "Display Entity furniture crashes old clients"
```yaml
furniture:
  detect_viabackwards: true
```

### "Players mining bedrock into void"
```yaml
bedrockbreak:
  disable_on_first_layer: true
```

### "Vanilla tripwire conflicts with custom blocks"
```yaml
stringblock:
  disable_vanilla_strings: true
```
