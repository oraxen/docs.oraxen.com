# Custom Armors

Oraxen lets you create custom armor sets with unique textures when equipped. There are three methods available, each designed for different Minecraft versions.

## Quick Comparison

| Method | MC Version | Base Material | Shader Compatible | Recommended |
|--------|------------|---------------|-------------------|-------------|
| [Components](/creating-content/armors/components) | 1.21.2+ | Any | ✅ Yes | ✅ Best choice |
| [Trims](/creating-content/armors/trims) | 1.20-1.21.1 | Chainmail | ✅ Yes | For 1.20-1.21.1 |
| [Shaders](/creating-content/armors/shaders) | 1.18-1.19.4 | Leather | ❌ Needs CIT | Legacy servers |

## Choosing the Right Method

### Use Components for:
- **All 1.21.2+ servers** - This is the optimal choice
- **Any armor material** - Not restricted to leather or chainmail
- **Full shader compatibility** - Works with Optifine, Iris, etc.

### Use Trims for:
- **1.20-1.21.1 servers** that don't support Components
- **Shader compatibility** without needing extra mods
- **Non-leather materials** - Uses chainmail by default

### Use Shaders for:
- **Pre-1.20 servers** or servers allowing older clients
- **Legacy configurations** that already use this method

<Callout type="warning">
**Shader Method Limitations**

The shader method breaks with Optifine/Iris shaders. Oraxen generates CIT textures to fix this for Optifine, but Iris users need [CIT Resewn](https://modrinth.com/mod/cit-resewn) installed.
</Callout>

## Feature Comparison

| Feature | Components | Trims | Shaders |
|---------|------------|-------|---------|
| Any base material | ✅ | ❌ Chainmail | ❌ Leather |

| Trim patterns | ✅ | ✅ | ❌ |
| Dyeable | ✅ | ❌ | ✅ |
| Performance | ✅ Best | ✅ Good | ⚠️ Okay |

## Basic Configuration

All armor methods share a similar base configuration:

```yaml
my_helmet:
  displayname: "<gradient:#4B36B1:#6699FF>My Helmet"
  material: DIAMOND_HELMET  # Or LEATHER_HELMET for shaders
  Pack:
    generate_model: true
    parent_model: "item/generated"
    textures:
      - my_helmet.png
  # Method-specific configuration below
```
